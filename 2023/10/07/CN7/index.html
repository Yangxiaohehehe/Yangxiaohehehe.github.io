Intro

名称：MAC（Medium Access Control）属于Data link layer，接近于physical layer，
作用：用来确定多路访问信道下一个使用者的协议

背景：网络分类有两种传播方式，点对点和广播。广播信道有时候也称为多路访问信道( multiaccess channel) 或者随机访问信道

内容：

  • Multiple access protocol
  • A real system(LAN): Ethernet
  • DLL SWitching（网桥和交换机）

广播网络的problem

Data communication的方式有三种：

  • Unicast(一对一)
  • Broadcast（一对everone 局域网用这个）
  • Multicast（一对一组）
广播信道有时被称为多址信道或随机存取信道。
广播的问题：如何分配一个broadcast channel 到多个竞争的用户当中（因为这是一个共享的信道）

MAC协议

Def：determine who goes next on a multiaccess channel（belong to sublayer of the data link layer called the MAC(Medium Access Control) sublayer）（就是决定谁进入channel）

There are two methods of allocating channels:

  • Static allocation
    • the channel is like a circuit – only one person is allowed to use it.
    • unused bandwidth will be lost （wasted
  • Dynamic allocation
  • the channel is open, with some computers being able to access unused bandwidth from others.
  • there is no dedicated bandwidth

Static allocation

在多个竞争用户之间分配单个信道的传统做法是把信道容量拆开分给多个用户使用，

方法：FDM TDM

FMD什么情况下有效：当用户数量固定且少量的时候，并且每个用户都有沉重的(缓冲的)流量负载时

FDM的问题：

  • 后如果目前对通信感兴趣的用户少于N个，则部分频谱将被浪费。
  • 如果超过N个用户想要通信，其中一些用户将被拒绝许可
  • 即使用户的数量是N且恒定的，当一些用户处于静止状态时，其他人也不能使用他们的带宽，所以这只是浪费。
  • 对于突发数据流量(峰值流量指流量比为1000:1)，分配的小子通道大部分时间是空闲的，无法处理峰值流量。

FDM的数学计算时延： 在一个容量为C bps 的信道上发送一帧所需要的平均时延T 。假设，随机到达帧的平均到达率为 λ帧／秒，帧的长度可变，其均值为每帧 1/µ位。利用这些参数，可以算出信道的服务率为µC 帧／秒。 标准排队理论的结果是T ＝1/(µC-λ) ,这样的结果只有在不存在信道竞争时才成立

现在我们将单个信道分成N个独立的子信道，每个子信道的容量为 C/N bps 。现在，每个子信道的平均到达率变成λ/ N。重新计算 ，我们得到
Tn=NT(带入就能算到的)

Dynamic Channel Allocation

静态的问题：所有的传统静态信道分配方法都不适应突发性的流量，于是得动态
五个假设：

  • Station Model
  • Single Channel Assumption
  • Collision Assumption
  • Continuous/Slotted Time(连续/开槽时间)
  • Carrier/No Carrier Sense

Station Model

模型由N个独立台站组成(也称为终端)。

每个站生成传输帧。

一旦生成了一个帧，该站就被阻塞，在帧成功传输之前什么也不做，也称为流量独立

Single channel Assumption

核心内容

一个单信道对所有communication开放

就硬件而言，所有工作站都是等效的。

但是协议软件可以给不同的电台分配优先级。

Collision Assumption

如果两个帧同时传输，它们将“碰撞”->在碰撞中，两个帧都完全丢失。->发生碰撞的帧必须稍后重新传输

所有台站都能探测到碰撞。除了碰撞之外没有其他错误。

Continuous/Slotted Time

Continuous time:时间被视为一个连续体。时间不是被分割成离散的时间间隔。帧传输可以在任何时刻开始。


Slotted Time：时间被分成离散的间隔(时隙)。帧传输总是从一个时隙的开始处开始。一个槽可以包含0、1或更多帧。分别对应于空闲的时间槽、一次成功发送，或者一次冲突。

Carrier/No Carrier Sense

载波侦昕或不昕（ Carrier Sense or no carrier sense ）。有了载波侦昕的假设，一个站
在试图用信道之前就能知道该信道当前是否正被使用。如果信道侦昕结果是忙，则没有一
个站会再去试图使用该信道。如果没有载波侦昕，站就无法在使用信道之前侦昕信道，它
们只能盲目地传输，以后再判断这次传输是否成功

Multiple access protocol （多路访问协议）

也叫作Random Access Protocol（随机访问协议）:

特点：使用渠道竞争，可能导致碰撞

典型随机存取协议：ALOHA，CSMA和CSMA/CD


受控访问协议（Controlled Access）： 特点：是分配信道，没有碰撞

ALOHA Protocol

Two version： Pure ALOHA 和Slotted ALOHA

纯ALOHA

basic idea：

想发就发，发送方总是可以通过监听信道输出来发现它的帧是否被销毁，如果帧被销毁，发送方只是随机等待一段时间（必须得随机的时间），然后再发送一次

可以随时发送数据，如果同时有2个以上的台站发送数据，碰撞是不可避免的

Pure ALOHA的数学描述

帧时”（frametime ）来表示传输一个标准的、固定长度的帧所需要的时间（即帧的长度除以比特率

假设：根据泊松分布生成新的frame：
每帧时新增帧数(用户生成):N(平均值) （N>1就会冲突）
每帧时新增帧数(信道生成):G(平均值)
G >= n;低载G=N(无碰撞)，重载G>N(重传)
显然G>=N,在负载较低的情况下（即 N<O ），冲突很少发生，因此重传也很少，于是 G<N
在负载较高时，将会有很多冲突，所以 G>N.

在给定的一个“帧时”内希望有G 帧，但生成K帧的概率服从泊松分布：
Pr[k] = G^k e^-G / k! ( (一个帧时内信道中产生k个帧，泊松分布))
Pr[k=0] = e^-G （一个帧时内信道中产生0个帧）

ALOHA 信道的效率：
和吞吐量S有关:吞吐量 就是负载乘以成功传输的概率 Po一S=GP0
在两个“帧时”长的问隔中，生成帧的平均数是 2G(Dangerous period of collision ：2t)
因此，在整个易受冲突期中，不发送帧的概率是 Po=e^-2G 。利用 S=GPo ，则可以得到：
S=Ge-2G
最大的吞吐量出现在当 G=0.5 时， = 1/2e,

分槽ALOHA

背景：能将 ALOHA 系统的容量增加一倍的方法）。他的建议是将时间分成离散的间隔，这种时间间隔称为时间槽（ slot),
每个时间槽对应于一帧。这种方法要求用户遵守统一的时间槽边界。取得同步时间的一种办法是由一个特殊的站在每个间隔起始时发出一个脉冲信号，就好像一个时钟一样。

特点：站不允许用户每次敲入回车键就立即发送帧。相反，它必须要等到下一个时槽的开始时刻。
　　　一旦发送开始一个frame，就不会有冲突
　　　碰撞只能发生在间隔(槽位)的最开始。

分槽的效率：
在测试帧所在的同一个时间槽中没有其他流量的概率是 e^-G ，于是可以得到：
S=Ge^-G
尖峰在 G=l 处，此时吞吐量为S=l/e

对比

分槽效率高是两倍
分槽的危险时间是一个帧时，pure是两个

Pure ALOHA:
A Frame is sent at once when it is generated
Collision may be occurred all the time

Slotted ALOHA
Frame is sent only at the very beginning of interval
Frame is sent successfully once no collision is occurred at the beginning of interval (slot)

Carrier Sense Protocols

Def：CSMA：Carrier Sense Multiple Access，
特点：先听后发，是ALOHA的提升

Types：
Non-persistent CSMA
Persistent CSMA：
　　1-persistent CSMA
　　P-persistent CSMA

Non-persistent CSMA

Basic idea：
先感知信道，如果没有其他人在发送，那么它就开始发送
如果通道已经在使用中，它不会感知它，而是随机等待一段时间并重复

优点：随机等待一段时间可以减少碰撞的概率
缺点:延迟较长(随机时间，可能没有数据传输)

Persistent CSMA (1-persistent)

1－坚持载波检测多路访问
Def: 当一个站有数据要发送时，它首先侦听信道，确定当时是否有其他站正在传输数据：如果信道空闲，它就发送数据。否则，如果信道忙，该站等待直至信道变成空闲：然后，站发送一帧。如果发生冲突，该站等待一段随机的时间，然后再从头开始上述过程。这样的协议之所以称为 1－坚持，是因为当站发现信道空闲时，它传输数据的概率为 l

优点:延迟比非持久性短
问题:如果两个或两个以上的电台同时等待，一旦通道空闲，碰撞是不可避免的。

P-persistent CSMA

Basic idea:
1.一个站点感知信道，如果没有其他人发送，那么它以概率p发送，并延迟一个单位时间以概率(1-p)发送。
2.如果信道忙，站点等待并持续感知，一旦信道空闲，重复1
3.如果某电台延迟了1个单位时间的传输，重复1

So, 1-persistent is a special example of p-persistent

How to compute collision window

问题：在传播过程中，因为时延τ存在，A检测空闲发送了，在τ内，B检测空闲，也发送，就导致了collision
Assume：signal’s propagation speed v is about:
v = 200m /µs，NIC delay is tPHY，then：t= S/v
Collision window (slot time)= 2t + 2tPHY

If repeater is used, assume delay caused by repeater is t_中继器, then:
Collision window (slot time)= 2*(t + tPHY+ N×t中继）

CSMA/CD(多了个collision detection)

Basic idea：

  1. 一个电台感知信道，如果没有其他人在发送，那么它就开始发送。
  2. 当信道处于繁忙状态时，工作站等待并持续感知，当信道处于空闲状态时，立即发送数据。
  3. 如果发生碰撞，中止传输，随机等待一段时间，然后重复传输

特点：
1.各站在发送信号的同时接收到自身的信号，因此，如果信号不同，则检测到碰撞
2.检测到碰撞时立即中止传输，并发送阻塞信号，以便在发生碰撞时通知其他站点;所有电台都需要一个随机时间来重新发送
3.广泛应用于局域网的MAC子层
多了个碰撞检测：就是变发送边真挺，也就是检测信道上电压的变换情况

CSMA/CD通道可以处于三种状态之一：
争用
传输
闲置

Collision detection and processing:

  1. 假设数据在最远的两个终端之间传播的时间长度为τ。(单程传播时延)
  2. 如果第二个终端在τ - ε时开始传输，那么发送方注意到碰撞所需的最大时间是2τ - ε。
  3. 最小时间量是两者同时开始发射的时间，使最小时间量简单地称为τ。

A Bit-Map Protocol

basic idea：
每个竞争期正好包含N个槽。如果0号站有一帧数据要发送，则它在第 0个槽中传送1位。在这个槽中，不允许其他站发送。不管 0号站做了什么，1号站有机会在1号槽中传送1位，但是只有当它有帧在排队等待时才这样做。一般地，j号站通过在j号槽中插入 j位来声明自己有帧要发送。当所有N个槽都经过后，每个站都知道了哪些站希望传送数据。这时候，它们便按照数字顺序开始传送数据了（N个站，每个站有0到N-1的地址）

过程:
每个争用周期由恰好N个时隙组成，其中一个时隙时间至少为2τ。
如果第i站(0<= i <= N -1)有帧要发送，它在第i个时隙发送1位;否则，在第i个槽位传输0位。
当所有的时隙都通过后，电台开始按数字顺序发送信号。
在最后一个就绪站传输完它的帧后，另一个n位争用周期开始。

Performance analysis
假设:时间单位为争用位槽，由d个时间单位组成的数据帧，N为站数或槽数。
low load: d/(N+d)
high load: d/(d+1)
Mean frame delay (high load): N(d+1)/2

缺点：没有优先级

Binary Countdown (二进制倒数)

背景：基本位图协议存在一个问题。每个站的开销是 位，所以该协议不可能很好地扩展到含有上千个站的网络中

方法：通过使用二进制的站地址，我们可以做得更好。如果一个站想要使用信道，它就以二进制位串的形式广播自己的地址，
从高序的位开始。假定所有地址都有同样的长度。不同站地址中相同位在同时发送时被信道布尔或 BOOLEAN OR ）在一起。我们把这样的协议称为二进制倒计数 binary countdown ）协议，它曾经被用在 Datakit 中（ Fraser, 1987 ）。它隐式地假设传输延迟可忽略不计

basic idea：
1.每个站点都有一个二进制地址。所有地址长度相同。
2.为了传输，电台以二进制位串的形式广播它的地址，从高阶位开始。
3.来自不同电台的每个地址位置的位是布尔或一起(所谓的二进制倒计时)。
4.一旦工作站看到其地址中的高阶位位0被1覆盖，它就放弃。
5.在获胜的站点传输了它的帧之后，没有可用的信息告诉有多少其他站点要发送，所以算法从下一帧开始。
简而言之：就是所有的二进制地址进行or，如果发现自己高位的0是1了，就放弃

Efficiency analysis：
N站地址需要的位是log2N
效率:d / （d +log2N)
如果帧的第一个字段是地址，那么效率是100%
