What is operating system

Computer(计算机组成和两种mode)

在现代操作系统中，CPU实际上都在两种截然不同的模式中花费时间



Computer=Hardware + OS + user Programs

User mode:在用户模式下，代码没有对硬件的直接控制权限，也不能直接访问地址的内存。程序是通过调用系统接
口(System APIs)来达到访问硬件和内存。在这种保护模式下，即时程序发生崩溃也是可以恢复的。在你的电
脑上大部分程序都是在用户模式下运行的。


Kernel mode:在内核模式下，代码具有对硬件的所有控制权限。可以执行所有CPU指令，可以访问任意地址的内存。
内核模式是为操作系统最底层，最可信的函数服务的。在内核模式下的任何异常都是灾难性的，将会导致整
台机器停机。

OS 的特点

  • 在kernel mode 中运行
  • 对所有硬件都具有控制权
  • 可以执行任何机器能执行的指令
  • huge complex long-lived

OS的major task

  • provide abstractions
  • manage resources

Provide abstractions

主要就是程序在执行的时候很复杂很原始，人很难懂，然后OS就会

隐藏这些实现的细节，把他们抽象化让作者能读懂。

在不同的物理机器上都可以执行相同的东西。

Manage resource

Def：OS高效的分配资源以使得多用户or多程序可以共同执行

例如：进程管理 内存管理 文件管理 设备管理这四个

目的：

  • Time multiplexing(比如让CPU决定哪一个先执行)
  • space multiplexing（内存和disk决定fairness 和protection等）

#History of Operating System

Content

一共四代：
  1. 第一代（45-55）：使用机器语言 难以理解 无操作系统 只有数字的计算
  2. 第二代（55-65）：有了汇编语言和FORTRAN 有一些OS 科学计算
  3. 第三代（65-80）：除此之外有了C等语言 有了UNIX操作系统 科学计算和一些应用
  4. 第四代（80-now）：更多的语言 更多的操作系统 科学计算和广泛应用

第二代

主要是有了汇编语言和OS

OS: FMS IBSYS

Application: scientific and engineering calculation

第三代

相较于第二代 开始了multiprogramming（多程序）和timesharing（多用户）

key idea: 保持CPU和IO busy不断有任务

第四代

LSI、芯片、电路的发展，os进入个人电脑

Computer Hardware Review

CPU 内存 IO设备 通过总线连接并互相交互

Processor（处理器）

步骤：Fetch（取址）（从内存捕捉命令送到IR然后PC指向下一个命令）->Decode（译码）（IR中的命令被decoder解释）->Execute->Fetch



Pipeline：流水线操作 多指令共同进行节约时间 流水线技术是一种将每条指令分解为多步，并让各步操作重叠，从而实现几条指令并行处理的技术。程序中的指令仍是一条条顺序执行，但可以预先取若干条指令，并在当前指令尚未执行完时，提前启动后续指令的另一些操作步骤。



Superscalar CPU：超标量CPU

流水线技术是一种将每条指令分解为多步，并让各步操作重叠，从而实现几条指令并行处理的技术。程序中的指令仍是一条条顺序执行，但可以预先取若干条指令，并在当前指令尚未执行完时，提前启动后续指令的另一些操作步骤。

Memory

memory一般包括:register, cache, main memory, disk, tape. 容量越小 操作速度越快 越贵

寄存器（Registers）

与CPU同材料 无延迟 寄存器是最贴近CPU的，而且CPU只与寄存器中进行存取。

（寄存的意思是，暂时存放数据，不中每次从内存中取，它就是一个临时放数据的空间，火车站寄存处就是这个意思）

Cache（缓存）

Cache hit 和miss， 且分L1Cache和l2 Cache



因为如果老是操作内存中的同一址地的数据，就会影响速度。于是就在寄存器与内存之间设置一个缓存。


一级缓存速度明显高于二级缓存，因为价格的原因就设置了二级缓存

Main memory（主存）

成分有RAM ROM EEPROM等

Disks（磁盘）

组成：磁盘组合由一个或多个圆盘组成，他们围绕一根中心主轴旋转，圆盘的上下表面涂抹了一层磁性材料，二进制位被存储在这些磁性材料上。其中，0和1在磁材料中表现位不同的模式。



I/O devices

有三种方式do input/output

Programmed I/O (PIO)

可编程的I/O指的是含有I/O指令的计算机程序，在该程序中有一个循环，会一直停留在循环直到I/O设备向CPU发送准备好数据传输的信息，这是一个让CPU处于不必要的繁忙的I/O operation。

Interrupt I/O

由于上述的可编程I/O让CPU处于不必要的繁忙之中，所以出现了中断驱动的方法，通过中断功能和特殊命令来通知接口，只要I/O设备有了需要的数据，便会发出中断请求信号。同时CPU可以继续执行其他程序。中断控制器会监控I/O设备是否准备好进行传输数据，并发出中断请求信号给CPU，检测到中断信号，CPU会停下来处理I/O传输。

Direct Memory Access (DMA)

这个方法让数据的传输不需要CPU的干预，直接允许I/O设备使用内存总线，数据传输的完成与否由DMA控制器监控，并且与CPU进行交互。

Process（进程）

Def：进程（Process）是计算机中的程序关于某 数据集 合上的一次运行活动，是系统进行 资源分配 的 基本单位 ，是操作系统结构的基础。

两点：1. a program in execution 2. a container that holds all the information needed to run a program



Process Table

内容：存储关于processes的信息和PCB（process control block）（是用于记录UID PID GID）

Process Tree

就是一个process里面可能有他的子process


不同process之间的协同合作的交流称为IPC（inter-process communication）

Address Space

所谓进程地址空间（process address space），就是从进程的视角看到的地址空间，是进程运行时所用到的虚拟地址的集合。




特点：

  • 一个进程没法看另一个的地址空间
  • 即使两个进程有相同的指针地址，他们也指向内存中不同的区域
  • 可以动态地更改映射

Difference between a process and a program

  1. Processes are dynamic,while programs are static.A process has a certain life cycle,and a program is a collection of instructions;
  2. A process contains both the program code and the resources needed to run the program.

Files

特点：

  • 文件系统提供一个抽象模型用于保护独立的文件
  • A directory（文件夹）用于对分件一起进行分组
  • 路径名可以找到文件位置
  • 根文件夹是文件夹分层制度的顶级
  • 每一个进程都有一个当前工作文件夹（PWD process working directory）

Protection

在UNIX的文件被9-bit的二元保护码保护

Shell

Def: A shell is a user interface for access to an operating system’s services (Shell 是一个命令解释器，它在操作系统的最外层，负责直接与用户进行对话，把用户的输入解释给
操作系统，并处理各种各样的操作系统的输出结果，输出到屏幕反馈给用户。这种对话方式可以是交互式也可以是非交互式的)

System calls

Def：在a running program和OS中间的界面（一般用汇编语言）.

操作系统内核为应用程序提供的服务和函数。用户通过调用系统的服务和函数完成相应的功能的过程称为系统调用。


background

  • 运行过程不确定
  • 结果不可再现
导致需要对运行过程加以干涉相互制约
