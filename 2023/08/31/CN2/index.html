How to design

Design Issues

  • Addressing
  • Error Control
  • Flow control
  • routing
  • Multiplexing and de-mutiplexing

Layered Model

OSI

Structure： 7 tiers

Physis->Data Link->Newtwork->transport->Session->Presentation->Application



Advantage：减少复杂性，标准化的接口可以交换信息，模块化工程不影响相互，确保互操作性，加速进化，容易教和学。

Application

Network processes to application
就是应用，提供网络服务比如电子邮件 文件传输等

Presentation

Data Representation

保证接受系统数据的可读性，数据的形式结构等

Session（对话）

Interhost communication

建立 应用 管理应用层之间的对话

Transport

End-to-end connections

信息realiable传输，建立维护终结虚拟电路，控制信息流，关心host之间的数据传输问题

Network

Addresses and best path

在两端系统中提供连接和路径选择，路由领域

Data Link

Access to media

通过介质提供可靠的信息传输，物理地址，网络，拓扑，错误分析等

Physical

Binary transmition

转接头的大小，电压，材料这些光电性质

TCP/IP Model

Structure: Network Interface->Internet->Transport->Application

Protocol

不同层之fatkun间有自己的协议，这是一个水平的概念

协议更像一种运行的程序

单位：DDL是帧（frame），网络层是数据包（Packet），传输层是Segment（数据段），物理层是二进制（bit）

这个网站介绍了所有的协议：
https://zhuanlan.zhihu.com/p/477662535

Service

服务是一个竖直的概念，下一层给上一层提供服务。

Encapsulation（封装）

发出信息的人从顶层开始逐层向下，封装就是将数据或函数等集合在一个个的单元中（我们称之为类）。被封装的对象通常被称为抽象数据类型。

Standardization

在网络发展的早期，各个互联网公司都有自己的一套标准，在某些功能和接口上并不统一，兼容性很差。IBM公司首先推出了系统网络体系结构SNA，随后DEC公司宣布了自己的数字网络体系结构DNA，这就是两套不同的标准。


许多组织发挥了很有用的作用：美国国家标准化协会（ANSI），
电气电子工程协会（IEEE），
国际电信联盟（ITU），
国际标准化组织（ISO），
Internet协会（ISOC）和相关的Internet工程任务组（IETF），
电子工业联合会（EIA）和相关的通信工业联合会（TIA）
