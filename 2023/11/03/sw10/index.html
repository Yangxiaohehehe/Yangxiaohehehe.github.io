Intro

Def：Formal specification is part of a more general collection of techniques that are known as ‘formal methods’
是形式化方式的一种，基于数学和逻辑的定义描述

包括：
1.Formal specification
2.Specification analysis and proof
3.Transformational development（需求转化为设计代码等）
4.Program verification（自动的测试验证）

目标：
Complete
Consistent一致性
Concise
Unambiguous
Valid—state exactly what the user wants

Semantics

DEf：means‘meaning’

Formal Semantics：Meaning expressed in mathematics

Formal semantic model:Complete semantic definition of a language in mathematics（离散数学）

Formal semantics permit dependable communication between all parties（自然语言容易引起无解，但是计算机不会，不会引起差异性）

Kinds of OF language

procedural（过程性的）：
　　计算由计算机要执行的预期操作序列定义
　　大多数high-level language都是这样的
Declarative：
　　计算由计算机应处于的理想状态定义
　　很多规范化语言是这样的

区别：
D：关于What而不是HOW
P：关于How而不是What
举例：
图书馆例子：
一本书既可以在书库中，也可以在备用书库中或借出。
如果一本书在书库中或在预留本上，那么就可以申请
我们希望
　　将概念和陈述形式化
　　证明一些定理，以确信规范是正确的

Resolution Refutation（归结法）

Def：一种算法，让机器在一种逻辑上推理证明
解析原理：描述了一种用最少的替换在条款数据库中找出矛盾的方法。

解析反驳：证明定理的方法是否定待证的语句，并将否定的目标添加到已知或假定为真的公理集合中。（遵循反证法原则）

然后使用推理的解析规则来证明这导致了矛盾

步骤：

1.将前提或公理转化为子句形式
　　子句形式没有并没有括号没有箭头等价等，只有or和非
　　转换方法：一句转化为多个子句 要保证多个子句同时true
2.在公理集合中加入要以分句形式证明的内容的否定项
3.将这些分句放在一起进行解析，产生新的逻辑分句
4.通过生成空子句来产生矛盾
5.用来产生空子句的替换是那些被否定目标的反面为真的替换

例子见PPT吧

FOrmal method

功能：提供软件质量，降低错误率

only a few software developed with formal methods：
1.Successful software engineering
2.Market changes
3.Limited scope of formal methods
4.Limited scalability of formal methods

Formal specification
　Come after the system requirement
　tight feedback loop between detailed requirement specification and the formal specification

Formal specification languages

Algebraic approach：
　System is described in terms of operations and their relationships
　larch,OBJ,Lotos

Model-based approach：
　A model of the system is built using mathematical constructs such as sets and sequences, and the system operations are defined by how they modify the system state
　Z,VDM,B,CSP,还有p-net

Interface Specification

大型系统被分解成若干子系统，这些子系统之间有明确定义的接口

子系统接口的规范允许独立开发不同的子系统

接口可定义为抽象数据类型或对象类

形式化规范的代数方法特别适合接口规范

sub-system interface specification
